# OctoAcme Role Onboarding Checklist

## Purpose
This checklist helps new team members quickly onboard to their project role and understand their responsibilities, collaboration patterns, and key resources.

---

## General Onboarding (All Roles)

### Access & Tools
- [ ] GitHub repository access granted
- [ ] Project board access granted
- [ ] Communication channels joined (Slack, Teams, etc.)
- [ ] Calendar invites received for key meetings
- [ ] Access to documentation repository and knowledge base

### Context & Documentation
- [ ] Read [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Review [Roles and Personas](octoacme-roles-and-personas.md) document
- [ ] Review current project documentation in docs/ folder
- [ ] Understand project lifecycle phases and current project phase
- [ ] Review project one-pager and success metrics

### Team Introductions
- [ ] Meet with Project Manager for role overview
- [ ] Meet with direct collaborators based on role
- [ ] Attend daily standup and team meetings
- [ ] Understand escalation paths and decision-making process

---

## Role-Specific Onboarding

### Project Sponsor
- [ ] Review strategic alignment and organizational priorities
- [ ] Understand budget and resource constraints
- [ ] Review milestone approval process
- [ ] Schedule regular steering committee meetings
- [ ] Identify key escalation scenarios

### Business Analyst
- [ ] Review existing requirements documentation
- [ ] Meet with key stakeholders for context
- [ ] Understand requirements elicitation process
- [ ] Access requirements management tools
- [ ] Review backlog and acceptance criteria format
- [ ] Shadow requirements workshop if possible

### QA Lead
- [ ] Review existing test strategy and test plans
- [ ] Access test automation frameworks and tools
- [ ] Understand CI/CD pipeline and quality gates
- [ ] Review defect tracking process
- [ ] Meet with developers to understand testability practices
- [ ] Review release readiness criteria

### Communications Specialist
- [ ] Review stakeholder map and communication plan
- [ ] Understand stakeholder communication preferences
- [ ] Access communication distribution channels
- [ ] Review messaging templates and brand guidelines
- [ ] Schedule recurring status update cadence
- [ ] Identify upcoming communication milestones

### Risk Manager
- [ ] Review current risk register
- [ ] Understand risk assessment methodology (impact/likelihood)
- [ ] Schedule recurring risk review meetings
- [ ] Meet with Project Sponsor to understand risk tolerance
- [ ] Identify key risk areas for the project
- [ ] Review contingency plans and mitigation strategies

### Developers
- [ ] Set up local development environment
- [ ] Review coding standards and PR conventions
- [ ] Understand branching strategy
- [ ] Access CI/CD pipeline documentation
- [ ] Review test coverage requirements
- [ ] Shadow code review process

### Product Manager
- [ ] Review product vision and roadmap
- [ ] Understand prioritization framework
- [ ] Access user research and metrics
- [ ] Review backlog and upcoming features
- [ ] Meet with stakeholders and customers
- [ ] Understand release planning process

### Project Manager
- [ ] Review project charter and objectives
- [ ] Understand timeline and milestones
- [ ] Access project tracking tools and dashboards
- [ ] Review risk register and dependencies
- [ ] Understand budget and resource allocation
- [ ] Schedule kickoff for ongoing coordination

---

## First Week Goals by Role

### Project Sponsor
- Understand project strategic value and ROI
- Identify any immediate resource or organizational barriers
- Confirm alignment with organizational priorities

### Business Analyst
- Complete stakeholder interviews
- Review existing requirements for gaps
- Identify requirements to be clarified or refined

### QA Lead
- Assess current test coverage
- Identify quality risks and gaps
- Draft initial test strategy updates if needed

### Communications Specialist
- Validate stakeholder map is current
- Send initial introduction and status update
- Plan next 2-3 communications milestones

### Risk Manager
- Review and update risk register
- Identify 2-3 top risks requiring immediate attention
- Schedule first risk review meeting

### Developers
- Complete first code contribution or bug fix
- Participate in code review
- Understand current sprint goals

### Product Manager
- Review and refine top 3 backlog priorities
- Meet with key stakeholders
- Understand current customer feedback

### Project Manager
- Deliver first status update
- Identify any immediate blockers
- Ensure team rhythm is established

---

## 30-Day Onboarding Goals

- [ ] Fully autonomous in day-to-day role responsibilities
- [ ] Established working relationships with key collaborators
- [ ] Contributing to project deliverables
- [ ] Comfortable with project tools and processes
- [ ] Able to identify and escalate issues appropriately
- [ ] Understand how success is measured for your role

---

## Onboarding Feedback

After 30 days, provide feedback to the Project Manager:
- What went well in onboarding?
- What could be improved?
- Are there gaps in documentation or process?
- What additional support would be helpful?

---

_This checklist should be customized based on project needs and team size._
